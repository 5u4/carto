本页是 carto 所假定的词汇表：承载全部结构的**清单（manifest）**、让失效可被检测的
**新鲜度（freshness）**模型、以及在重构中幸存的 **`carto:` 链接**系统。这里的一切都由
`@carto/core` 强制执行——正是 [](carto:cli) 所运行的那份代码。

## 心智模型

四个理念层层嵌套。**清单**（`carto.json`）承载一个**节点**列表；每个节点带 `id`、`slug`、
`parent`，以及一个**源文件**列表。一个源文件通过**内容哈希**（外加它 sync 时的 git 提交）
把节点系到一个文件上。把那个存储哈希与文件当前字节比较，就得出节点的**新鲜度**——
`fresh` / `unsynced` / `stale` / `missing`。与这一切正交地，一个 **`carto:` 链接**用节点
不可变的 `id` 寻址，渲染器再把它变成由节点 slug 链得出的 URL——因此链接永远不指向一个会
移动的位置。

## 清单

每个文档根一份 `carto.json` = 一个站点。它只承载结构和源追踪——**不含正文**。schema
（`packages/core/src/schema.ts:24`）要求 `version: 1`、一个非空且唯一的 `locales` 列表且
`defaultLocale` 是其成员、以及一个 `nodes` 数组。可选的顶层 `home` 指定站点根 `/` 跳转到的
节点 id；省略它则根路径回退到第一个根节点。跨字段规则在 `superRefine` 中检查：重复的
locale、`locales` 之外的 `defaultLocale`、以及重复的节点 id 都会失败
（`packages/core/src/schema.ts:33`）。

一个节点有四个字段（`packages/core/src/schema.ts:17`）：

- **`id`** —— 必填、全局唯一、不可变。模式 `^[a-z0-9][a-z0-9-]*$`
  （`packages/core/src/schema.ts:3`）——小写、数字、连字符；无 `.`、无 `/`。这是链接目标。
  永不重命名。
- **`slug`** —— 可选的展示/URL 段；默认取 id（`packages/core/src/tree.ts:9`）；必须在同级中
  唯一。改起来很便宜——链接指向 id，而非 slug。
- **`parent`** —— 可选的父节点 id。作为根节点时**省略该键**；不要写 `parent: null`
  （schema 把它类型化为可选字符串）。不存在的 parent 是警告而非错误——你可以从树的中间开始生成。
- **`sources`** —— 该页面所描述行为所在的文件。**只写 `file`**，一个相对于 `carto.json`
  所在目录的路径，且不含 `..` 段（`packages/core/src/schema.ts:5`）；`carto sync` 同时填入
  `hash`（内容指纹）和 `commit`（sync 时的 git `HEAD`）。schema 拒绝没有 `hash` 的
  `commit`（`packages/core/src/schema.ts:18`）。对纯定位页面，该数组可为空。

`sources` 是**失效的十字准星**：太宽会触发虚假的“stale”抖动；太窄会让真实变更溜过。只注册
一个节点真正解释的承重文件。它的反面是覆盖——`coverageReport` 遍历文档根、减去每个被追踪的
源文件、返回未被追踪的余下部分（`packages/core/src/coverage.ts:60`），即 `carto coverage`
所打印的内容。

## 新鲜度 = 哈希 vs. 现实

每个源文件存一个文件字节的 16 字符 sha256（`packages/core/src/hash.ts:5`）。`carto sync`
重新计算并写入它；`carto status` 重新计算并比较。比较得出四种状态之一
（`packages/core/src/status.ts:5`）：

- **fresh** —— 存储哈希 == 实际哈希。
- **unsynced** —— 该源文件尚无存储哈希（你加了 `file` 但没跑 `sync`）
  （`packages/core/src/status.ts:45`）。
- **stale** —— 存储哈希 != 实际哈希：文件在上次 sync 之后变了。
- **missing** —— 文件在磁盘上不再存在（`packages/core/src/status.ts:42`）。

一个节点的状态取其各源文件中**最差**的一个，顺序为 `fresh < unsynced < stale < missing`
（`packages/core/src/status.ts:21`，在 `packages/core/src/status.ts:32` 归约）。因为它基于
哈希而非 git，无论用什么版本控制都能抓住任何编辑——但一个子系统里*新增*的文件在你注册它并
重新 sync 之前是不可见的。

### 完整示例——一次真实的失效转变

`carto status` 在 sync 之后立即报告每个节点 `fresh`。给一个被追踪的源文件追加一个字节，
其所属节点就翻转为 `stale`——这正是仓库的 `pnpm e2e` 冒烟测试对
`packages/core/src/hash.ts`（`concepts` 节点的一个源文件）所做的，随后再恢复它：

```
$ carto status
fresh     concepts

$ printf '\n' >> packages/core/src/hash.ts
$ carto status          # exits non-zero
stale     concepts

$ carto sync && carto status
synced 6 node(s)
fresh     concepts
```

## `carto:` 链接

在页面之间用**逻辑 id** 链接，绝不用文件路径或 slug。在 `.mdx` 里，写一个目标为
`carto:<id>` 的 Markdown 链接：

- `carto:<id>` —— 按 id 指向一个节点。
- `carto:<id>#<anchor>` —— 指向该节点内的一个标题锚点。
- 一个目标为 `carto:<id>` 的**空标签**——构建时填入目标在当前语言下的标题。

解析器解析目标，拒绝保留的联邦 `/` 形式（不支持），并在清单中查找该 id
（`packages/core/src/resolver.ts:41`）；未知 id 在 `carto validate` 时是错误。URL 本身是
从根到节点的 slug 链，非默认语言带语言前缀（`packages/core/src/tree.ts:30`）——这正是为什么
移动一个节点永不断裂链接。

**代码锚点不同。** 正文中的 `path:line` 提及（如 `packages/core/src/hash.ts:5`）把读者指向
源码。在 MVP 中它渲染为纯文本——无永久链接——并且与 `sources`（机器追踪的失效集）**分离**。
实践中两者指向同一段承重代码。
