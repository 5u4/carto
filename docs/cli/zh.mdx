`carto` 二进制是七个子命令（`packages/cli/src/index.ts:13`）。每个都是 `@carto/core`
之上一层薄薄的、确定性的封装——它做哈希、做校验、或启动渲染器；它从不写正文。所有命令都从
当前目录读取 `carto.json`，因此**在文档根里运行它们**。在本仓库里那就是仓库根，用
`pnpm exec carto <cmd>` 调用。

## 概览

你按一个大致的顺序运行它们。`init` 一次性搭好文档根；然后你和技能一起编写文档；`sync`
写入哈希；`status` 报告新鲜度，`coverage` 找出还没有任何节点追踪的文件；`validate` 跑
完整检查；`dev` 或 `build` 预览或渲染站点。两个命令**写入**（`init`、`sync`）；四个是
**只读检查或预览**（`status`、`coverage`、`validate`、`dev`）；一个**产出静态站点**
（`build`）。`status`、`coverage`（配合 `--fail-on-uncovered`）和 `validate` 用退出码
表示失败，因此可以直接接进 CI。

| 命令 | 为你做什么 | 退出码 |
|---|---|---|
| `init` | 搭建 `carto.json` + `docs/` + `carto.config.mjs` | 若 `carto.json` 已存在则为 1 |
| `status` | 报告每个节点的新鲜度 | 有任一节点非 fresh 则非零 |
| `sync` | 写入每个源文件哈希 | 有源文件缺失则为 1 |
| `coverage` | 列出没有任何节点 source 追踪的文件 | 0，或配合 `--fail-on-uncovered` 为 1 |
| `validate` | 全面检查：schema、树、同步、链接 | 有任何错误则为 1 |
| `dev` | 在 localhost:4321 预览站点 | 透传自 Astro |
| `build` | 把静态站点渲染到 `dist-site/` | 透传自 Astro |

## `carto --help`

```
$ carto --help
Generate and maintain carto documentation (carto)

USAGE carto init|status|sync|coverage|validate|dev|build

COMMANDS

      init    Scaffold carto.json and docs/ in the current directory
    status    Report each node's freshness
      sync    Recompute and write every source hash
  coverage    Report files not covered by any node source
  validate    Validate schema, tree, sync state, and links
       dev    Preview the site for the current doc root
     build    Build the static site for the current doc root
```

## init

搭建一个空的 `carto.json`、一个 `docs/` 目录，以及一个 `carto.config.mjs` Starlight
覆盖 stub（仅当其不存在时）。拒绝覆盖已存在的清单——护栏是一个 `exists` 检查，命中则
退出 1（`packages/cli/src/commands/init.ts:16`）。`--locales en,zh` 声明语言列表，
`--defaultLocale` 选定默认语言，`--codeRoot` 让 sources 指向一个相对文档根解析的代码树
（`packages/cli/src/commands/init.ts:11`）。

```
$ carto init
initialized carto.json, docs/, carto.config.mjs (locales: en)

$ carto init
carto.json already exists; refusing to overwrite
```

## status

每个节点打印一行 `state id`——该状态取其各源文件中**最差**的一个
（`fresh` < `unsynced` < `stale` < `missing`）——只要有节点非 fresh 就以非零退出
（`packages/cli/src/commands/status.ts:28`）。这是你的 CI 新鲜度闸门，也是你挑选 `refresh`
目标的方式。每个非 fresh 的源文件还会打印它上次 sync 时的 git 提交——
`stale a.ts (was 5514fce…)`——刷新时做 `git diff` 的基准
（`packages/cli/src/commands/status.ts:22`）。

```
$ carto status
fresh     overview
fresh     getting-started
fresh     skill
fresh     cli
fresh     concepts
fresh     internals
```

## sync

唯一的确定性写操作：重新计算并写入每个源文件哈希，给每个源文件盖上当前 git `HEAD` 戳，
并刷新 `updated_at`（`packages/cli/src/commands/sync.ts:13`）。这个提交戳正是后续
`carto status` 报告的 diff 基准；在 git 仓库之外它会被直接省略
（`packages/cli/src/git.ts:5`）。在任何 `carto.json` 编辑之后、以及代码变化之后运行它。
若某个已注册的源文件不存在，它会中止（`packages/core/src/manifest.ts:91`）。

```
$ carto sync
synced 6 node(s)
```

## coverage

只读检测：列出文档根下没有任何节点 `sources` 追踪的文件——新代码引入的孤儿
（`packages/cli/src/commands/coverage.ts:19`）。它尊重 `.gitignore` 和 `.cartoignore`，
并始终排除 carto 自身的产物（`carto.json`、`docs/`、`dist-site/`）、`.git/` 和
`node_modules/`（`packages/core/src/coverage.ts:12`）。它从不生成——决定为某个孤儿写文档
始终是一次刻意的调用。默认退出 0；传 `--fail-on-uncovered` 让它成为一道 CI 闸门，有未追踪
文件时退出 1（`packages/cli/src/commands/coverage.ts:28`）。

```
$ carto coverage
covered   24 / 24 files (100.0%)

all files covered
```

## validate

完整的闸门：schema、id/slug 唯一性、parent 环、同步状态、每种语言一个 `.mdx`、`home`（若设置）
指向一个真实节点，以及每个 `carto:` 链接是否可解析
（`packages/cli/src/commands/validate.ts:17`）。缺失的文档、unsynced/stale 的节点，以及未知的
`home` 都是错误；悬空的 parent 只是警告。有问题时以退出码 1 结束，并为每个问题打印一行
`error:`；干净时打印 `validate: ok`（`packages/cli/src/commands/validate.ts:66`）。

```
$ carto validate
validate: ok
```

在页面尚不存在时，同一个命令会列出它发现的缺口：

```
$ carto validate
error: missing doc: docs/overview/en.mdx
error: missing doc: docs/overview/zh.mdx
...
```

## dev

以 dev 模式启动内置的 Astro + Starlight 模板，传入 `CARTO_ROOT=$PWD`，使它物料化并服务
*你的*文档根（`packages/cli/src/commands/dev.ts:25`）。入口页是
`http://localhost:4321/overview/`——默认语言（`en`）无前缀；`zh` 位于 `/zh/overview/`。
用 Ctrl-C 停止它。

## build

同一个模板，生产模式：把静态站点渲染到文档根下的 `dist-site/`（已被 gitignore）
（`packages/cli/src/commands/build.ts:7`，它在 `packages/cli/src/commands/dev.ts:15`
调用 `runTemplateScript('build')`）。
