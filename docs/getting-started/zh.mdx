---
title: 快速开始
---

目标：从一个没有文档的目录，走到一份**通过校验的 carto 文档集**，并渲染成站点。结束时，
你将已经搭好文档根、驱动你的 agent 走完生成循环，并看到 `carto validate` 退出码为 0。
下面的走查就是生成*本*站点的真实运行，附带真实输出。

## 前置条件

- **一个 LLM/agent**，能读文件、写文件——carto 是 BYO-LLM。agent 读代码，写 `.mdx` 页面
  和清单（[](carto:skill)）；carto 从不臆造正文。
- **PATH 上的 `carto` CLI。** 在本仓库里它是一个 workspace 二进制：执行
  `pnpm install && pnpm build && pnpm install`（第二次 install 才链接该 bin——见仓库
  `README.md`）之后，用 `pnpm exec carto` 调用它。
- **Node ≥ 20。**

## 你即将运行的循环

这个循环是一条直线加一条回边。`carto init` 搭好文档根；agent 读代码并运行 `carto status`；
你规划节点树（id、parent、sources）；为每个节点 × 语言写 `docs/<id>/<locale>.mdx`；用只写
`file` 的 sources 编辑 `carto.json`；运行 `carto sync` 填入每个哈希；然后 `carto validate`。
validate 要么把你送回去修某个页面，要么退出码为 0，你就完成了。

需要掌握三个概念：一个**节点**是一个页面，带一个不可变的 `id`；一个**源文件**是节点所描述
行为所在的文件（失效的十字准星）；一个**语言（locale）**是一种语言——每个节点在每个已声明的
语言下都需要一个 `.mdx`，否则 `validate` 失败（`packages/cli/src/commands/validate.ts:65`）。

## 步骤

### 1. 搭建文档根

`carto init` 在当前目录写入一个空的 `carto.json` 和一个 `docs/` 目录，并在尚不存在
`carto.config.mjs` 时额外写入一个 Starlight 覆盖 stub。它拒绝覆盖已存在的清单
（`packages/cli/src/commands/init.ts:16`）：

```
$ carto init
initialized carto.json, docs/, carto.config.mjs (locales: en)

$ carto init
carto.json already exists; refusing to overwrite
```

传 `--locales en,zh` 可在一开始就声明多种语言，或传 `--codeRoot ../src` 让 sources
指向文档根之外的代码树（`packages/cli/src/commands/init.ts:11`）。

### 2. 让 agent 规划并写作

带一个范围（一个目录或一组文件）调用 [](carto:skill)。agent 读代码，选定一棵节点子树
（id、parent、sources），并为每个节点写 `docs/<id>/<locale>.mdx`。然后它编辑 `carto.json`
把每个节点连同其 sources 注册进去——**只写 `file`，不写 `hash`**（`carto sync` 会填）。
清单形态见 [](carto:concepts)。

### 3. 先 sync，再 validate

`carto sync` 计算并写入每个源文件的哈希，并刷新 `updated_at`
（`packages/cli/src/commands/sync.ts:12`）。`carto validate` 随后校验 schema、树、同步状态，
以及每个 `carto:` 链接是否可解析（`packages/cli/src/commands/validate.ts:19`）。

## 完整示例——为本仓库编写文档

这是生成这套六节点自文档的真实运行，从仓库根（即文档根）执行：

```
$ pnpm exec carto sync
synced 6 node(s)

$ pnpm exec carto status
fresh     overview
fresh     getting-started
fresh     skill
fresh     cli
fresh     concepts
fresh     internals

$ pnpm exec carto validate
validate: ok
```

`sync` 填入了 6 个节点的哈希；`status` 报告每个节点 `fresh`（存储哈希 == 实际哈希）并退出 0；
`validate` 打印 `validate: ok`。用 `carto dev` 在 `http://localhost:4321/overview/` 预览，
或用 `carto build` 渲染静态站点——两者都在 [](carto:cli) 里讲。

## “完成”是什么样

`carto validate` 退出码为 0，且 `carto status` 显示每个节点 `fresh`。若某个源文件之后发生变化，
该节点就变成 `stale`——对它重新调用技能（`refresh` 模式）并重新 `sync`。这个失效信号正是全部
意义所在；它如何工作见 [](carto:concepts)。
