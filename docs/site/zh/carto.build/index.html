职责

src/build.ts 是 carto 哈希流水线与 Astro Site Template 之间的桥梁。它的任务有两重：计算哪些节点已陈旧，然后外调 Astro 来渲染静态站点——并通过环境变量把陈旧集合交给模板。它只暴露一个函数 build，由 CLI Commands 的 build 子命令调用。

关键类型与函数

  • BuildOptions——输入契约：

    export interface BuildOptions {
  irPath: string;
  contentDir: string;
  outDir: string;
}

  • build(options)——读取 IR，推导出陈旧节点 id，派生 Astro 进程，并在子进程干净退出时兑现。

数据流

  1. 读取 + 校验。 readIr(options.irPath) 通过 IR & Source I/O 边界加载并按 schema 校验 IR。

  2. 比对。 diffHashes(ir, await currentHashes(ir, options.contentDir)) 将每个节点存储的哈希与磁盘上的实时哈希比较，返回发生变化的源文件列表。

  3. 归约为陈旧 id。 将这些变化映射为 { anchor, file }，交给 staleNodes（来自 Hashing Core），并将每个陈旧节点的 id 收集进 staleIds。

  4. 定位模板与 Astro 可执行文件。 模板目录相对已编译模块解析——resolve(dirname(fileURLToPath(import.meta.url)), "..", "astro")——因此无论当前工作目录为何，它都指向同级的 astro/ 模板目录。Astro 可执行文件通过 require.resolve("astro/package.json") 找到，再拼接到 bin/astro.mjs。

  5. 派生进程。 spawn(process.execPath, [astroBin, "build", "--root", templateDir], …) 以当前 Node 二进制的子进程身份启动 astro build，采用 stdio: "inherit"，使 Astro 的输出直接流向用户终端。三个环境变量把 carto 的状态带入构建：

    • CARTO_CONTENT_DIR——绝对内容目录（resolve(options.contentDir)）。
    • CARTO_OUT_DIR——绝对输出目录（resolve(options.outDir)）。
    • CARTO_STALE_IDS——以换行连接的陈旧节点 id（staleIds.join("\n")）。

  6. 等待子进程。 借助 Promise.withResolvers，函数将子进程的生命周期接到一个 promise 上：

    • child.on("error", reject)——派生失败则 reject。
    • child.on("close", (code, signal) => …)——退出码 0 兑现（settle()）；非空 signal 以 astro build terminated by signal <signal> reject；任何其他非零退出码以 astro build exited with code <code> reject。

    因此返回的 promise 恰好在 Astro 构建成功时兑现，在任何失败模式下 reject。

child.on("close", (code, signal) => {
  if (code === 0) {
    settle();
  } else if (signal !== null) {
    reject(new Error(`astro build terminated by signal ${signal}`));
  } else {
    reject(new Error(`astro build exited with code ${code ?? "null"}`));
  }
});

关系

  • 由 CLI Commands 的 build 子命令调用。
  • 复用 Hashing Core（diffHashes、staleNodes）与 IR & Source I/O（readIr、currentHashes）来计算陈旧度。
  • 通过三个 CARTO_* 环境变量喂给 Astro Site Template——模板读取它们以加载内容并标记陈旧页面。
  • 在渲染时落实 Staleness Lifecycle 的报告一侧。
